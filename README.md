## ⚙️ 2. 자료구조 및 알고리즘 적용 계획

### 요약 테이블

| 번호    | 데이터 / 기능             | 자료구조 / 적용 알고리즘       | 비고                                                                      |
| :------ | :------------------------ | :----------------------------- | :------------------------------------------------------------------------ |
| `DS-01` | 사용자별 팔로우/팔로잉 관계 | **그래프 (인접 리스트)** | 사용자를 정점(Vertex), 팔로우 관계를 간선(Edge)으로 표현하여 관계 관리에 용이 |
| `DS-02` | 일일 섭취 영양소 관리     | **해시맵 (HashMap)** | 특정 영양소(Key)의 섭취량(Value)을 `O(1)` 시간 복잡도로 빠르게 조회 및 수정      |

| `Algo-03` | 챌린지 달성도 정렬        | **듀얼 피벗 퀵 정렬** | 사용자의 챌린지 달성도에 따라 순위를 매겨 조회                            |

### 🔎 Algo-01. 음식 정보 검색 (자동 완성)

> **"방대한 음식 DB에서 사용자가 원하는 음식을 빠르고 정확하게 검색할 수 있도록 지원합니다."**

* **📄 상세 내용**
    * 사용자가 식단 작성 시, '닭가'처럼 음식 이름의 일부만 입력해도 '닭가슴살', '닭갈비' 등 해당 문자열을 포함하는 음식 목록을 신속하게 제시하여 사용자 편의성을 극대화합니다.

* **💡 적용 알고리즘**: **KMP (Knuth-Morris-Pratt)**
    * 불일치가 발생했을 때 이동할 다음 위치를 미리 계산한 **실패 함수(π 함수)**를 사용하여 불필요한 비교를 건너뛰는 고효율 문자열 검색 알고리즘입니다.
    * 이를 통해 전체 텍스트를 재탐색하지 않고, 검색어 내에서 효율적으로 이동하며 검색을 수행합니다.

* **✨ 기대 효과**
    * ✅ **빠른 속도**: 단순 비교 방식(`O(N*M)`)보다 월등히 빠른 `O(N+M)`의 시간 복잡도를 보장하여, 빠른 응답 속도를 유지할 수 있습니다.
    * ✅ **UX 향상**: 사용자가 수많은 데이터 중에서 원하는 항목을 쉽게 찾을 수 있어 사용자 경험이 향상됩니다.

* **🤔 고려할 점**
    * KMP는 '김치찌게'와 같은 오타나 띄어쓰기 차이를 처리하지 못합니다. 이를 보완하기 위해 검색어 전처리 또는 **유사도 검사 알고리즘(예: 편집 거리)** 추가 도입을 고려할 수 있습니다.

### ❤️‍🩹 Algo-02. 사용자 맞춤 음식 추천

> **"사용자의 건강 상태와 목표에 가장 적합한 음식을 추천하여 개인 맞춤형 건강 코칭을 제공합니다."**

* **📄 상세 내용**
    * 회원 정보(예: 당뇨 전단계, 다이어트 목표)를 기반으로 음식 DB의 각 항목에 대해 **'추천 점수'**를 계산합니다.
    * 예) 당뇨 전단계 사용자에게는 당류에 높은 페널티를, 식이섬유에는 가산점을 부여합니다.
    * `추천 점수 = (w1*단백질) - (w2*당류) - (w3*나트륨) + ...`
    * 계산된 점수를 **퀵 정렬** 등 효율적인 알고리즘으로 정렬하여 상위 N개의 음식을 추천합니다.

* **✨ 기대 효과**
    * ✅ **신뢰성 있는 정보 제공**: 개인의 건강 상태에 맞는 정확한 정보를 제공하여 서비스의 핵심 목표를 달성합니다.
    * ✅ **서비스 차별화**: '나에게 딱 맞는 음식'이라는 구체적인 가치를 제공하여 타 서비스와의 차별점을 만듭니다.

* 🤔 **고려할 점**
    * 매 추천 시마다 전체 DB를 계산하면 성능 저하가 발생할 수 있습니다. 카테고리별로 **1차 필터링 후 알고리즘을 적용**하거나, **캐싱 전략**을 도입하여 응답 속도를 개선해야 합니다.

### 🏆 Algo-03. 챌린지 달성도 정렬

> **"챌린지 참여자의 순위를 실시간으로 보여줌으로써 강력한 동기를 부여합니다."**

* **📄 상세 내용**
    * '하루 1만보 걷기' 챌린지 참여자들의 현재 걸음 수를 기준으로 실시간 순위를 매깁니다.
    * 달성도는 숫자 데이터이므로, 빠르고 효율적인 정렬이 핵심입니다.

* **💡 적용 알고리즘**: **듀얼 피벗 퀵 정렬 (Dual-Pivot Quicksort)**
    * Java의 `Arrays.sort()`에서 기본 데이터 타입 정렬에 사용하는 알고리즘입니다.
    * 2개의 피벗을 사용해 데이터를 3부분으로 분할하므로, 기존 퀵 정렬보다 평균 성능이 우수하며 `O(n log n)`의 빠른 속도를 보여줍니다.

* **✨ 기대 효과**
    * ✅ **참여 유도**: 실시간 순위를 통해 경쟁심과 재미를 유발하여 사용자의 지속적인 참여를 이끌어냅니다.
    * ✅ **안정적인 성능**: 다수의 사용자가 참여하는 챌린지에서도 성능 저하 없이 안정적인 순위 정보를 제공할 수 있습니다.

* 🤔 **고려할 점**
    * **동점자 처리 기준**이 명확해야 합니다. '목표 달성 선착순'과 같은 2차 정렬 기준을 정의해야 합니다.
